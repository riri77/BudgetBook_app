## 家計簿Webアプリ開発における技術概要

本プロジェクトでは、ユーザーの資産管理を効率的かつ安全に実現するため、**PythonのWebフレームワークであるFlask**と、AWSが提供するクラウドサービス群を組み合わせてシステムを構築します。

## Flaskの利用理由と役割

- Flaskは軽量で拡張性に優れたPython製のWebフレームワークであり、API開発に適しています。
- 本アプリケーションのバックエンドとして、ユーザー認証のトークン検証、資産データのCRUD（作成・読み取り・更新・削除）処理、ビジネスロジックの実装に用います。
- RESTful APIを設計し、フロントエンドや他のサービスとの柔軟な連携を可能にします。

## AWSサービスの採用と役割

- **AWS Cognito**
    
    ユーザーのサインアップ、サインイン、認証管理を担い、高度なセキュリティ機能（多要素認証やパスワードリセットなど）を提供します。Flask APIと連携して認証情報の検証を行います。
    
- **AWS Amplify**
    
    フロントエンドのホスティングおよびバックエンドの連携基盤として利用可能で、開発・デプロイを効率化します。
    
- **Amazon RDS（MySQL/PostgreSQL）または DynamoDB**
    
    ユーザー情報や資産履歴、閲覧申請情報などの永続化データを管理します。RDSはリレーショナルデータベースの強みを活かした設計が可能で、DynamoDBはスケーラブルなNoSQLデータベースとして利用できます。
    
- **AWS CloudWatchなどの監視ツール**
    
    システムの健全性とパフォーマンス監視に用い、障害時の迅速な対応を支援します。
    

## システム構成のメリット

- クラウドサービスを活用することで、スケーラビリティ・可用性・セキュリティが確保されます。
- Flaskの柔軟なAPI設計により、将来的な機能追加や他システム連携が容易です。
- AWSのマネージドサービスを利用することで、認証・データベース管理・デプロイ運用の負荷を軽減し、開発効率を高めます。
